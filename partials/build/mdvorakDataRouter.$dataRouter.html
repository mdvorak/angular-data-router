$dataRouter
service in module mdvorakDataRouter

Description

Centerpiece of the data router. It tracks the $location and loads the main view data.

Properties

  • api

    Its here to make your life easier.

    Returns
    Object

    Reference to the $dataApi instance.

  • current

    Note that you should in most cases use $dataResponse object in the controller to work with the data instead of this reference.

    Returns
    Object

    Currently loaded response for the main view is available here. See $dataResponse for its signature.

  • navigate

    Navigates to resource URL. See $dataRouter.url() for more details.

  • navigate

    Allows changing of the current view url. It does not get, however, refected in parent view in any way, if applicable.

  • registry

    Its here to make your life easier.

    Returns
    Object

    Reference to the $dataRouterRegistry instance.

  • reload

    Reloads data at current location. If content type remains same, only data are refreshed, and $routeUpdate event is invoked on $dataResponse object. If content type differs, full view refresh is performed (that is, controller is destroyed and recreated).

    If you refresh data, you must listen to the $routeUpdate event on $dataResponse object to be notified of the change.

  • url

    Gets or sets current view resource URL using $dataApi.url().

    If the url is not in the configured API namespace, error is logged and nothing happens.

    Returns
    String

    Resource URL that is being currently viewed.

Events

  • $routeChangeError

    This event is broadcasted when the view or data failed to load and there is no error view configured (or error view failed to load). There is not much you can do at this point, it probably means you have configuration error in your application.

    Type:

    broadcast

    Target:

    root scope

  • $routeChangeStart

    This event is broadcasted whenever main view is about to change. The change can be averted by setting preventDefault on the vent.

    Type:

    broadcast

    Target:

    root scope

  • $routeChangeSuccess

    This event is broadcasted whenever main view has changed and the data are loaded. View is ready to be displayed.

    Type:

    broadcast

    Target:

    root scope
