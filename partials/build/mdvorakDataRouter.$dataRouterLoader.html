$dataRouterLoader
service in module mdvorakDataRouter

Description

Abstraction of data loading and view preparation. It uses views registered in $dataRouterRegistry.

Methods

  • normalizeMediaType(mimeType)

    Normalizes the media type. Removes format suffix (everything after +), and prepends application/ if there is just subtype.

    Parameters
    ParamTypeDetails
    mimeTypeString

    Media type to match.

    Returns
    String

    Normalized media type.

  • prefetchTemplate(mediaType)

    Eagerly fetches the template for the given media type. If media type is unknown, nothing happens. This method returns immediately, no promise is returned.

    Parameters
    ParamTypeDetails
    mediaTypeString

    Media type for which we want to prefetch the template.

  • prepareView(url, current, forceReload)

    Prepares the view to be displayed. Loads data from given URL, resolves view by its content type, and then finally resolves template and all other resolvables.

    Parameters
    ParamTypeDetails
    urlString

    URL of the data to be fetched. They are always loaded using GET method.

    current
    (optional)
    		Object

    Current response data. If provided and forceReload is false, routeDataUpdate flag of the response may be set, indicating that view doesn't have to be reloaded.

    forceReload
    (optional)
    		boolean

    When false, it allows just data update. Without current parameter does nothing.

    Returns
    Object

    Promise of completely initialized response, including template and locals.
